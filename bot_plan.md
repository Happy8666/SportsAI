# План разработки Telegram-бота для спортивной аналитики (скриншот → прогноз)

## Цель
Создать Telegram-бота, который анализирует скриншоты футбольных матчей и выдаёт статистику команд, прогноз исхода, ключевые факторы и уровень уверенности. Бот должен работать полностью бесплатно, для личного использования.

## Требования к функциональности

### Команды бота
- `/start` – приветствие, краткая инструкция.
- `/help` – полный список команд и примеры использования.
- `/predict <команда1> <команда2>` – ручной ввод команд для прогноза.
- **Отправка фото** – бот распознаёт команды через OCR и автоматически делает прогноз.
- `/history` – показывает последние 5 запросов пользователя с датой и прогнозом.

### Обработка данных
1. **Распознавание скриншотов** (OCR):
   - Использовать **EasyOCR** с предобработкой через **OpenCV** (повышение контраста, бинаризация, удаление шума).
   - Извлекать названия команд из текста.
   - Если распознавание не удалось – предложить ввести команды вручную.

2. **Получение статистики**:
   - Использовать бесплатный API **Football-Data.org** (зарегистрироваться и получить ключ).
   - Реализовать поиск команды по названию, получение последних 5 матчей, статистику голов, историю встреч.

3. **Прогноз** (MVP):
   - На основе средних голов за последние 5 матчей, преимущества домашнего поля и истории встреч.
   - Рассчитывать вероятности победы хозяев, ничьей и победы гостей.
   - Вычислять уверенность прогноза (от 0 до 1) на основе разброса вероятностей.
   - Генерировать 3–5 ключевых факторов (форма, личные встречи, мотивация).

4. **Хранение истории**:
   - SQLite база данных с таблицей `history` (user_id, команды, прогноз, дата).
   - Логирование всех действий и ошибок в файл.

5. **Деплой**:
   - Развернуть на бесплатном хостинге (Render.com или Fly.io) с использованием переменных окружения.
   - Бот должен работать круглосуточно.

## Технический стек
- **Язык**: Python 3.10+
- **Фреймворк**: aiogram (асинхронный)
- **HTTP-клиент**: aiohttp (для API)
- **OCR**: EasyOCR + OpenCV
- **База данных**: SQLite (встроенная)
- **Формула прогноза**: простая эвристика (в дальнейшем можно заменить на ML)
- **Управление зависимостями**: pip + requirements.txt или Poetry
- **Конфигурация**: .env (токены, ключи API)
- **Логирование**: logging

## Этапы работы с использованием скиллов Superpowers

### 1. Установка Superpowers (если ещё не установлены)
Выполнить в терминале:
```bash
npm install -g opencode-superpowers
2. Активировать скилл superpowers и начать мозговой штурм
Использовать под-скилл brainstorming для уточнения архитектуры и выбора библиотек.

Определить, как лучше организовать асинхронность, обработку ошибок, кэширование API.

3. Создать детальный план (под-скилл writing-plans)
Разбить работу на задачи с приоритетами.

Оценить время на каждую задачу (но можно реализовать всё в одном сеансе).

4. Исполнить план (под-скилл executing-plans)
По очереди создать все файлы и написать код.

Следовать лучшим практикам: чистый код, комментарии, обработка исключений.

5. Проверить работоспособность (под-скилл verification-before-completion)
Протестировать каждую команду локально.

Убедиться, что история сохраняется.

6. Подготовить деплой
Создать Dockerfile (опционально) или инструкцию для Render/Fly.io.

Настроить переменные окружения на хостинге.

Задеплоить и проверить работу бота в production.

Требования к коду
Весь код должен быть на русском языке (комментарии, названия переменных на английском или русском – на ваше усмотрение, но комментарии лучше на русском).

Использовать аннотации типов.

Добавить docstrings для всех функций.

Обеспечить конфигурируемость через .env.

Логировать все важные события и ошибки.

Критерии готовности
Бот отвечает на /start, /help.

По команде /predict с двумя названиями команд возвращает прогноз.

При отправке фото распознаёт команды (хотя бы в 60% случаев) и возвращает прогноз.

Если команда не найдена или OCR не сработал – выводится понятное сообщение.

История запросов сохраняется и доступна по /history.

Код структурирован, есть логирование.

Бот запущен на бесплатном хостинге и доступен 24/7.
